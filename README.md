# 광운대역 도착 통학 생존 퀴즈

정보융합학부 2023204020 김우현

Streamlit 중간고사 대체 과제입니다.

이 프로젝트는 인천·경기권 출발지에서 광운대역까지 통학할 때의 경로 판단을 퀴즈로 풀어보는 웹 애플리케이션입니다. 단순히 노선 이름을 맞히는 방식이 아니라, 급행/일반열차, 환승역, 배차 리스크, 요금 부담, 소요 시간 범위를 함께 고려하도록 설계했습니다.

## 주요 기능

- 첫 화면에 제출자 정보 표시
- 로그인 및 회원가입 기능
- `@st.cache_data`를 활용한 JSON 데이터 캐싱
- 객관식, 단답형, 시간 추정형, 서술형 퀴즈
- 총점 계산 및 결과 유형 분류
- 지역별 정답률 차트와 문항별 해설 제공
- 교통 근거와 참고 출처 표시

## 프로젝트 구조

```txt
.
├── app.py              # Streamlit 앱 본문
├── main.py             # 실행 진입점
├── quiz_data.json      # 퀴즈 문항 데이터
├── requirements.txt    # 실행에 필요한 패키지
├── ScriptsMovie.txt    # 데모 영상 대본
├── assets/             # 퀴즈용 SVG 노선 이미지
└── .gitignore
```

`users.json`은 회원가입 정보를 저장하는 로컬 사용자 DB입니다. 앱 실행 시 파일이 없으면 기본 계정으로 자동 생성되며, 개인정보성 런타임 데이터이므로 GitHub에는 포함하지 않습니다.

## 실행 방법

1. 저장소를 내려받습니다.

```bash
git clone https://github.com/monitor5/streamlit_first.git
cd streamlit_first
```

2. 가상환경을 만들고 활성화합니다.

```bash
python -m venv venv
```

Windows PowerShell:

```bash
.\venv\Scripts\Activate.ps1
```

3. 필요한 패키지를 설치합니다.

```bash
pip install -r requirements.txt
```

4. Streamlit 앱을 실행합니다.

```bash
streamlit run main.py
```

## 기본 로그인 정보

```txt
아이디: student
비밀번호: 1234
```

회원가입 탭에서 새 계정을 만들 수 있습니다. 비밀번호는 평문이 아니라 SHA-256 해시값으로 `users.json`에 저장됩니다.

## 캐싱 적용 위치

이 프로젝트는 Streamlit의 `@st.cache_data`를 실제 데이터 로딩에 적용했습니다.

- `quiz_data.json` 로딩: 퀴즈 문항을 매번 다시 읽지 않고 캐시된 데이터를 재사용합니다.
- `users.json` 로딩: 사용자 데이터를 읽을 때 캐싱을 사용합니다.
- 회원가입 후 사용자 정보가 바뀌면 사용자 데이터 캐시를 비웁니다.
- 사이드바의 `퀴즈 데이터 캐시 새로고침` 버튼을 누르면 퀴즈 데이터 캐시와 입력한 답변 상태가 초기화됩니다.

## 퀴즈 구성

총 10문항으로 구성되어 있으며, 모든 문항의 도착지는 광운대역입니다.

- 객관식: 경로 판단, 환승 후보 비교
- 단답형: 핵심 환승역 입력
- 시간 추정형: 슬라이더로 현실적인 소요 시간 범위 맞히기
- 서술형: 통학 전략과 근거 작성

결과 유형은 총점에 따라 다음과 같이 나뉩니다.

| 점수 | 결과 |
| --- | --- |
| 0~4점 | 통학 입문자 |
| 5~7점 | 현실형 통학생 |
| 8~10점 | 광운대 통학 전략가 |
