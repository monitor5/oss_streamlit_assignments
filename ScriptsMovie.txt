Streamlit 프로젝트 데모 영상 대본

1. 첫 실행 화면

안녕하세요. 정보융합학부 2023204020 김우현입니다.
이번 프로젝트는 "광운대역 도착 통학 생존 퀴즈"입니다.
부평역, 가정역, 운정중앙역, 구리역, 동탄역처럼 서로 다른 출발지에서 광운대역까지 이동할 때 어떤 경로가 현실적인지 판단하는 퀴즈 앱입니다.

첫 화면 상단에는 과제 조건에 맞게 제 학부, 학번, 이름이 표시되어 있습니다.
앱은 Streamlit으로 만들었고, 메인 실행 파일은 main.py입니다.

2. 로그인 기능 설명

이제 로그인 기능을 시연하겠습니다.
기본 계정은 student, 비밀번호는 1234입니다.
로그인 폼에 아이디와 비밀번호를 입력하고 로그인 버튼을 누르면, 입력값이 users.json에 저장된 사용자 정보와 비교됩니다.
비밀번호는 그대로 저장하지 않고 해시값으로 저장되도록 만들었습니다.

로그인에 성공하면 퀴즈 화면으로 넘어가고, 실패하면 오류 메시지가 표시됩니다.
회원가입 탭에서는 새 아이디와 비밀번호를 입력해 사용자 계정을 추가할 수 있습니다.
회원가입 정보는 users.json에 저장되므로 서버를 껐다 켜도 유지됩니다.
단, users.json은 개인 사용자 데이터베이스이므로 GitHub에는 올리지 않도록 .gitignore에 추가했습니다.
앱을 처음 실행했을 때 users.json이 없으면 기본 계정이 자동으로 생성됩니다.

3. 캐싱 기능 설명

다음은 캐싱 기능입니다.
이 앱에서는 퀴즈 문항을 quiz_data.json 파일에 따로 저장해 두고, 앱 실행 중에는 이 JSON 데이터를 불러와서 사용합니다.
quiz_data.json을 읽는 함수에는 Streamlit의 @st.cache_data를 적용했습니다.
따라서 같은 데이터를 반복해서 읽을 때 매번 파일을 새로 처리하지 않고 캐시된 결과를 재사용합니다.

또한 사용자 데이터를 불러오는 함수에도 @st.cache_data를 적용했습니다.
회원가입으로 사용자 정보가 바뀌면 캐시를 비우고 users.json을 다시 읽도록 처리했습니다.

왼쪽 사이드바에는 "퀴즈 데이터 캐시 새로고침" 버튼이 있습니다.
이 버튼을 누르면 퀴즈 데이터 캐시가 지워지고, 이미 입력했던 정답과 결과 상태도 초기화됩니다.
이 부분이 캐싱 기능이 실제 앱 안에서 의미 있게 사용되는 지점입니다.

4. 퀴즈 구성 설명

이제 퀴즈를 풀어보겠습니다.
문항은 총 10개이고, 객관식, 단답형, 시간 추정 슬라이더형, 서술형이 섞여 있습니다.
모든 문항의 도착지는 광운대역으로 고정되어 있습니다.

이 앱은 단순히 노선 이름을 맞히는 퀴즈가 아니라, 실제 장거리 통학 상황을 가정합니다.
예를 들어 급행을 탈 수 있는지, 환승역을 어디로 잡을지, 시간이 빠른 경로와 비용이 낮은 경로 중 무엇이 더 현실적인지 판단해야 합니다.

각 문항에는 출발지, 도착지, 예상 소요 시간이 표시됩니다.
일부 시간 추정 문제는 정답을 바로 추측하지 못하도록 예상 시간을 물음표로 표시했습니다.
또한 "교통 근거 보기"를 열면 해당 문항을 만들 때 참고한 교통 정보와 출처를 확인할 수 있습니다.

5. 문항 풀이 예시

1번 문항은 부평역에서 광운대역까지 가는 경로 판단입니다.
부평역은 1호선과 인천1호선 환승역입니다.
급행을 이용할 수는 있지만, 광운대역까지 급행으로 바로 간다고 생각하면 안 됩니다.
그래서 급행은 서울 방향 일부 구간을 줄이는 용도로 보고, 이후 광운대 방면 일반열차 환승을 고려하는 선택지를 고르겠습니다.

2번 문항은 가정역 출발입니다.
가정역은 인천2호선 역이므로 광운대역까지 바로 갈 수 없습니다.
주안역에서 1호선으로 갈아타는 방법, 석남역과 온수역을 거치는 방법, 검암역에서 공항철도를 이용한 뒤 서울역에서 1호선으로 갈아타는 방법을 비교해야 합니다.

3번 문항은 단답형입니다.
가정역에서 인천2호선을 타고 1호선으로 바로 갈아탈 수 있는 대표 환승역은 주안역입니다.

4번과 5번 문항은 운정중앙역 출발입니다.
운정중앙역은 GTX-A를 쓰는 빠른 경로와, 경의중앙선으로 우회하는 저렴한 경로가 나뉩니다.
그래서 시간과 비용을 함께 고려해야 합니다.

6번과 7번은 시간 추정 슬라이더 문제입니다.
정확한 분 단위보다 현실적인 범위 안에 들어오는지를 평가합니다.
실제 통학 시간은 환승 대기, 배차 간격, 도보 이동, 혼잡도에 따라 달라지기 때문입니다.

마지막 10번은 서술형입니다.
운정중앙역의 빠른 경로와 저렴한 경로, 또는 가정역의 환승 후보 중 하나를 골라 본인의 통학 전략을 설명합니다.
비용, 시간, 배차, 환승 피로도 중 하나 이상을 근거로 적으면 참여 점수로 인정됩니다.

6. 결과 확인

모든 문항에 답한 뒤 "결과 보기" 버튼을 누르겠습니다.
앱은 각 문항을 1점씩 계산해서 총 10점 만점으로 점수를 보여줍니다.

객관식은 기준 답과 일치하면 점수를 줍니다.
단답형은 핵심 키워드가 포함되어 있으면 정답으로 처리합니다.
시간 추정형은 정답 범위 안에 들어오면 정답입니다.
서술형은 공백이 아니고 통학 판단 근거가 있으면 참여 점수를 줍니다.

결과 화면에서는 총점과 결과 유형이 표시됩니다.
0점에서 4점은 통학 입문자, 5점에서 7점은 현실형 통학생, 8점에서 10점은 광운대 통학 전략가로 분류됩니다.

아래에는 지역별 이해도 그래프와 문항별 확인 영역이 있습니다.
각 문항을 펼치면 내가 고른 답, 기준 답, 해설, 교통 근거, 출처를 확인할 수 있습니다.
이렇게 해서 단순히 맞고 틀린지만 보는 것이 아니라, 왜 그 경로가 현실적인지 복습할 수 있도록 만들었습니다.

7. 마무리

정리하면, 이 프로젝트는 Streamlit 기반의 통학 경로 판단 퀴즈 앱입니다.
과제 필수 조건인 첫 화면의 학번과 이름 표시, 로그인 기능, 캐싱 기능, 퀴즈 풀이와 결과 확인 기능을 모두 포함했습니다.
또한 문항 데이터를 JSON으로 분리하고, 사용자 데이터도 JSON 기반으로 관리하여 앱 구조를 명확하게 구성했습니다.

이상으로 데모를 마치겠습니다.
